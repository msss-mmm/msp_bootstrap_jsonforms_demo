# Reverse Proxy Configuration Guide

To ensure the Manufacturing Execution System correctly detects its subpath and handles absolute URLs, your external reverse proxy (Nginx) must be configured to pass the `X-Forwarded-Prefix` header.

## Recommended Nginx Configuration

When proxying to the application, add the `X-Forwarded-Prefix` header matching the location block:

```nginx
location /mes/ {
    proxy_pass http://your-backend-host:7001/;

    # Standard headers
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;

    # IMPORTANT: Pass the subpath prefix so the app can detect its location
    # If hosted at /mes/, set this to /mes
    proxy_set_header X-Forwarded-Prefix /mes;
}
```

### Why is this needed?

1.  **Frontend:** The internal Nginx uses this header to inject the correct base path into the HTML. This allows the Vue router to handle navigation correctly and assets to be loaded from the right location.
2.  **Backend:** Django uses this header (via a custom middleware) to adjust its `SCRIPT_NAME`. This ensures that any URLs generated by the backend (like for QR codes or redirects) include the correct subpath.

## Multiple Environments

This configuration allows you to host the *same* Docker container in multiple environments at different subpaths simultaneously:

- `http://hostname/mes` -> `proxy_set_header X-Forwarded-Prefix /mes;`
- `http://localhost:2000/prod` -> `proxy_set_header X-Forwarded-Prefix /prod;`

No rebuild or `.env` changes are required when moving between these locations!
